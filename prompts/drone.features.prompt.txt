== Platform constraints ==
DJI control surfaces work on Android only — every native call is gated by isDJIBridgeAvailable() / Platform.OS === 'android'. iOS shows the same UI but the SDK calls are no-ops; treat any drone command request from an iOS session as not-available. A "MOCK_MISSIONS=true" env flag swaps useWaypointMission for useMockWaypointMission so the UI can be tested without a real aircraft. Roles: only OPERATOR and ADMIN can issue drone commands (canControlDrone). GUEST is read-only.

== Connection & telemetry ==
Bridge: src/native/DJIBridge.ts.
Calls (all Promise<boolean> unless noted):
- djiIsSdkRegistered(): boolean — has the DJI SDK key been accepted by the device?
- djiRegisterSdk(): kicks the registration handshake.
- djiStartConnectionToProduct(): start product discovery; returns once the SDK begins linking.
- djiGetConnectionState(): DjiConnectionState. No params. Polled every 500–2000 ms by screens that need HUD telemetry. Fields: productPresent, productConnected, flightControllerConnected, remoteControllerConnected, cameraConnected, pilotLinkOk, sdkRegistered, onlyRcConnected, model (string), altitudeM, hSpeedMs, vSpeedMs, batteryPct, satelliteCount, aircraftLatitude, aircraftLongitude, distanceHomeM. Numeric fields are number | null (null = unknown).
- isDJIBridgeAvailable(): synchronous availability probe.
- djiSetPilotScreenOrientation(landscape: boolean): true = lock activity to landscape (pilot mode), false = release back to portrait.

Drone-location event stream:
- DJIFlightController emits 'DroneLocationEvent' with { latitude, longitude, heading (0–360 cw from N), altitude (m above takeoff) }.
- Subscribe via the React hook useFlightControllerLocation() (auto start/stop on mount/unmount).

== Flight controls (manual) ==
Prereq: pilotLinkOk === true and the aircraft has propellers ready / passes DJI safety checks.
- djiFlightStartTakeoff(): auto-takeoff to a safe hover.
- djiFlightStartLanding(): auto-land sequence in place.
- djiFlightStartGoHome(): return-to-home (RTH) sequence to the recorded home point.
- djiVirtualStickEnable(enable: boolean): enable virtual sticks before sending axes.
- djiVirtualStickSetAxes(roll: number, pitch: number, yaw: number, throttle: number): set the four control axes. Values are normalized stick deflections (-1…+1) in the app's pilot UI; the bridge maps them to DJI's degree/m/s ranges. Call only after djiVirtualStickEnable(true).

== Gimbal ==
- djiGimbalReset(): recenter the gimbal (axes that the aircraft supports).

== Camera ==
- djiCameraStartRecordVideo(): start video recording on the aircraft.
- djiCameraStopRecordVideo(): stop video recording.
- djiCameraTakePhoto(): single still capture.
- djiCameraContinuousZoom(direction: 1 | -1 | 0): 1 = zoom in, -1 = zoom out, 0 = stop. Only works on aircraft with a zoom lens.
- djiOpenDeviceGallery(): open the aircraft media gallery surface (vendor UI).
- djiGetCameraRecordingState(): { isRecording, cameraAvailable }.
- djiGetCameraOperationState(): { cameraAvailable, isRecording, isSwitchingMode, isShootingPhoto, cameraPipelineBusy, cameraBusy }. UI uses cameraBusy / isSwitchingMode / isShootingPhoto to gate buttons.
- djiDroneSettingsSetCameraMode(modeName: string): switch between camera modes (e.g. SHOOT_PHOTO, RECORD_VIDEO). Mode strings are the DJI SDK enum names.

== Drone safety / settings (Settings screen) ==
All return Promise<boolean>; persist on the aircraft itself.
- djiDroneSettingsGetSnapshot(): Record<string, unknown> — current values for everything below + read-only fields.
- djiDroneSettingsSetMaxFlightHeight(meters: number): hard altitude ceiling, typically 20–500 m.
- djiDroneSettingsSetGoHomeHeight(meters: number): RTH cruise altitude, must be ≥ obstacle clearance.
- djiDroneSettingsSetMaxFlightRadius(meters: number): geofence radius from home point.
- djiDroneSettingsSetMaxFlightRadiusLimitationEnabled(enabled: boolean): turn the radius geofence on/off.
- djiDroneSettingsSetLowBatteryWarning(percent: number): % battery for first warning (typical 20–50).
- djiDroneSettingsSetSeriousLowBatteryWarning(percent: number): % battery for forced RTH warning (must be lower than the regular threshold).
- djiDroneSettingsSetVisionSystemPowerOn(on: boolean): downward / forward vision sensors on.
- djiDroneSettingsSetCollisionAvoidanceEnabled(enabled: boolean): obstacle avoidance toggle (independent of vision power).

== Firmware ==
- djiDroneFirmwareGetAircraftStatus(): current firmware status object.
- djiDroneFirmwareCheckAircraft(): query latest firmware availability.
- djiDroneFirmwareStartAircraftUpgrade(): kick off the upgrade flow.
- djiDroneFirmwareUnsubscribeProgress(): stop receiving upgrade progress events.

== Waypoint mission (DJIMissionService) ==
File: src/native/DJIMissionService.ts. Used by the legacy Missions screen; wrapped by useWaypointMission hook.

WaypointPoint shape:
- latitude: number (WGS84 degrees)
- longitude: number (WGS84 degrees)
- altitude: number (metres above takeoff point)
- heading?: number (-180…180 degrees; aircraft yaw at this WP)
- cornerRadiusInMeters?: number (curved-path mode only)
- turnMode?: 'CLOCKWISE' | 'COUNTER_CLOCKWISE'
- gimbalPitch?: number (-90…0 degrees)
- speed?: number (m/s; 0 means use mission autoFlightSpeed)
- shootPhotoTimeInterval?: number (seconds)
- shootPhotoDistanceInterval?: number (metres)
- actions?: WaypointAction[]

WaypointAction shape:
- actionType: 'STAY' | 'START_TAKE_PHOTO' | 'START_RECORD' | 'STOP_RECORD' | 'ROTATE_AIRCRAFT' | 'GIMBAL_PITCH'
- actionParam?: number (ms for STAY, degrees for ROTATE_AIRCRAFT and GIMBAL_PITCH)

WaypointMissionConfig (passed to loadAndUploadWaypointMission):
- maxFlightSpeed?: number (m/s, [2, 15])
- autoFlightSpeed?: number (m/s, must be ≤ maxFlightSpeed)
- finishedAction?: 'NO_ACTION' | 'GO_HOME' | 'AUTO_LAND' | 'GO_FIRST_WAYPOINT' | 'CONTINUE_UNTIL_END'
- headingMode?: 'AUTO' | 'USING_INITIAL_DIRECTION' | 'CONTROL_BY_REMOTE_CONTROLLER' | 'USING_WAYPOINT_HEADING' | 'TOWARD_POINT_OF_INTEREST'
- flightPathMode?: 'NORMAL' | 'CURVED'
- gotoFirstWaypointMode?: 'SAFELY' | 'POINT_TO_POINT'
- repeatTimes?: number (1…255)
- exitMissionOnRCSignalLost?: boolean
- gimbalPitchRotationEnabled?: boolean
- waypoints: WaypointPoint[] (REQUIRED; minimum 2 waypoints to upload)

Defaults applied by useWaypointMission: autoFlightSpeed=8, maxFlightSpeed=15, defaultAltitude=50, finishedAction='GO_HOME', headingMode='AUTO', flightPathMode='NORMAL', gimbalPitchRotationEnabled=true.

Operations:
- DJIMissionService.loadAndUploadWaypointMission(config): validate + upload to aircraft. Drives state UPLOADING → READY_TO_EXECUTE.
- DJIMissionService.startWaypointMission(): start execution.
- DJIMissionService.pauseWaypointMission(): pause in place.
- DJIMissionService.resumeWaypointMission(): resume from current target index.
- DJIMissionService.stopWaypointMission(): cancel; aircraft holds position.
- DJIMissionService.setAutoFlightSpeed(speed: number): change cruise speed mid-flight (m/s, [2, 15]).
- DJIMissionService.getWaypointMissionState(): MissionState string.

States: 'DISCONNECTED' | 'NOT_SUPPORTED' | 'READY_TO_UPLOAD' | 'UPLOADING' | 'READY_TO_EXECUTE' | 'EXECUTING' | 'EXECUTION_PAUSED' | 'ERROR'.

Mission progress events: { type: 'upload' | 'execution' | 'executionStart' | 'executionFinish', uploadedWaypointIndex?, isSummaryUploaded?, targetWaypointIndex?, isWaypointReached?, error? }.

== Hotpoint / orbit (DJIHotpointService) ==
HotpointConfig:
- latitude: number (WGS84) — orbit centre.
- longitude: number (WGS84) — orbit centre.
- altitude: number (metres above takeoff, [5, 500])
- radius: number (metres; SDK firmware-dependent min/max — typical [5, 500]; reject 0)
- angularVelocity?: number (deg/s; default 20; max depends on radius — slower for larger radius)
- clockwise?: boolean (default true)
- startPoint?: 'NEAREST' | 'NORTH' | 'EAST' | 'SOUTH' | 'WEST' | 'CURRENT_HEADING' (default 'NEAREST')
- heading?: 'TOWARDS_HOTPOINT' | 'AWAY_FROM_HOTPOINT' | 'CONTROLLED_BY_RC' | 'USING_INITIAL_HEADING' (default 'TOWARDS_HOTPOINT')

Operations:
- DJIHotpointService.startHotpointMission(config)
- DJIHotpointService.pauseHotpointMission()
- DJIHotpointService.resumeHotpointMission()
- DJIHotpointService.stopHotpointMission()
- DJIHotpointService.setHotpointRadius(radius: number) — change radius mid-orbit
- DJIHotpointService.setHotpointAngularVelocity(angVel: number) — change speed mid-orbit
- DJIHotpointService.getState(): HotpointState

States: 'DISCONNECTED' | 'NOT_SUPPORTED' | 'READY_TO_EXECUTE' | 'INITIAL_PHASE' (flying to start point before orbit begins) | 'EXECUTING' | 'EXECUTION_PAUSED' | 'ERROR'.

Compatibility: Hotpoint is unsupported on Mavic Mini, Mavic Air 2 / 2S, M300 RTK.

== Follow Me (DJIFollowMeService) ==
FollowMeConfig:
- latitude: number — initial target position (typically phone GPS).
- longitude: number — initial target position.
- altitude: number — metres AMSL. If you cannot determine AMSL, ~30 m above the phone's GPS altitude is a safe default.
- heading?: 'TOWARD_FOLLOW_POSITION' (drone faces the user) | 'CONTROLLED_BY_REMOTE_CONTROLLER' (RC overrides yaw). Default 'TOWARD_FOLLOW_POSITION'.

Operations:
- DJIFollowMeService.startFollowMeMission(config)
- DJIFollowMeService.updateFollowTarget(lat: number, lon: number) — call ~1 Hz from a Geolocation.watchPosition callback to keep the drone tracking the moving phone.
- DJIFollowMeService.stopFollowMeMission()
- DJIFollowMeService.getState(): FollowMeState

States: 'DISCONNECTED' | 'NOT_SUPPORTED' | 'READY_TO_EXECUTE' | 'EXECUTING' | 'ERROR'.

Compatibility: Follow Me is unsupported on Mavic Pro (WiFi) and Spark.

== Permission prerequisites ==
- Android: ACCESS_FINE_LOCATION + ACCESS_COARSE_LOCATION granted (Map / Follow Me).
- iOS: DJI control is not available — refuse drone commands, but Map and Follow Me UI still need NSLocationWhenInUseUsageDescription.
- DJI bridge: requires the SDK key to be registered (djiIsSdkRegistered === true) before any product calls.

== Suggested call ordering for common flows ==
Manual takeoff: djiIsSdkRegistered → djiStartConnectionToProduct → wait for djiGetConnectionState.pilotLinkOk → djiFlightStartTakeoff → djiVirtualStickEnable(true) → djiVirtualStickSetAxes(...) → djiVirtualStickEnable(false) → djiFlightStartLanding (or djiFlightStartGoHome).
Waypoint flight: build WaypointMissionConfig (≥2 waypoints) → DJIMissionService.loadAndUploadWaypointMission(config) → wait for state READY_TO_EXECUTE → DJIMissionService.startWaypointMission() → optional pause/resume → on EXECUTING_FINISH the aircraft performs finishedAction.
Orbit a target: pick centre lat/lon, choose altitude / radius / speed → DJIHotpointService.startHotpointMission(config) → adjust mid-flight via setHotpointRadius / setHotpointAngularVelocity → DJIHotpointService.stopHotpointMission().
Follow phone: get phone GPS → DJIFollowMeService.startFollowMeMission({ latitude, longitude, altitude }) → on every Geolocation update call DJIFollowMeService.updateFollowTarget(lat, lon) → DJIFollowMeService.stopFollowMeMission() when done.
