== App identity ==
Argus is a React Native (iOS + Android) drone-ops app for security / surveillance teams. It pairs with DJI aircraft via the DJI Mobile SDK on Android (FlightController, Camera, Waypoint Mission Operator, Hotpoint, Follow Me). The backend is a NestJS + Supabase service. Each end user is scoped to a "deployment" — a site profile that owns its missions, public RTSP cameras, alerts and KPIs.

== Roles & permissions ==
Roles: GUEST, OPERATOR, ADMIN (legacy strings like client_admin / treycor_operator / viewer are normalised on the BE).
Permissions object carried on the User:
- canControlDrone   — OPERATOR + ADMIN issue SDK commands.
- canManageUsers    — ADMIN only (admin users screen).
- canEditMissions   — OPERATOR + ADMIN edit / toggle / duplicate / delete custom missions.
- canViewDashboard  — everyone (GUEST sees read-only dashboard).

== Deployment types ==
construction, commercial, school, sports, estate, residential. Each deployment has: name, location string, missions[], constraints { maxCustomMissions, canEditMissions, canToggleMissions }.

== Navigation ==
React Navigation native-stack, single Stack.Navigator at the root. initialRouteName = "Login". headerShown = false everywhere; each screen draws its own header (HamburgerMenu on the left, page title centre, action button right).

There is no Tab.Navigator. The "bottom tab bar" is a custom component <BottomNav /> that screens render at their bottom and that calls navigation.navigate(routeName). Bottom tabs:
1. Dashboard           → Dashboard
2. Live                → LiveView
3. Missions            → MissionsV2 (V2 Mission Center, NOT the legacy DJI planner)
4. Map                 → Map
5. Alerts              → Alerts

Auth flow:
Login → (Register / ForgotPassword as side branches) → Welcome → Discover or Deployment → Dashboard. After auth most navigation happens via navigation.push or via BottomNav.navigate.

Special: the Missions tab routes to MissionsV2. The legacy DJI Waypoint planner ("Missions" route) is reachable only from inside MissionsV2 by Launching the seeded "DJI Mission" entry.

== Cross-cutting flows ==

Auth: auth-context wraps Supabase login/refresh; access tokens are passed as Authorization: Bearer <token> from FE service helpers (apiRequest in lib/api-client). Logout clears persisted state.

Deployment: deployment-context holds the active DeploymentProfile + user. setDeployment writes to AsyncStorage key argus_next_deployment. Most lists (missions, alerts, public RTSP cams, dashboard KPIs) are scoped per deployment id.

Drone connect (Android only):
1. DJIBridge.isDJIBridgeAvailable() guards every native call.
2. djiGetConnectionState polled every 500–2000 ms updates HUD + connection-context.
3. drone-connection-context.connectedDroneIds[id] = true once flight controller is found.
4. Hardware link grace period (DRONE_LINK_GRACE_MS) avoids "no drone" modal flashing on cold start; a longer NO_DRONE_MODAL_AFTER_MS triggers the dismissible notice.

Mission lifecycle (legacy planner):
planning → upload waypoints → ready → start → executing → (pause ↔ paused) → stop / finish → finished. Errors set status=error and surface ErrorToast. Telemetry: HUD altitude, hSpeed, distanceHome, batteryPct, satelliteCount, droneLatitude/Longitude.

Custom missions (V2):
deployment-context exposes createMission, toggleMission, duplicateMission, deleteMission. New custom missions are stored on the BE under deployments/:id/missions; remainingCustomSlots = constraints.maxCustomMissions − customCount.

Public RTSP feeds:
GET /public-rtsp returns { byDeployment: { <DeploymentType>: PublicCamera[] } }; PUT replaces the full map. PublicCamera = { id, name, url, note? }. URL must be rtsp:// rtsps:// http:// or https:// (HLS/mp4). Reserved ids (drone, drone-cam, dronecam, system, system-drone, drone:*, system:*) are rejected by the service AND by a Postgres CHECK constraint; FE never sends these. Persistence: AsyncStorage key @argus/liveview_public_rtsp_by_deployment_v1; userTouchedPublicRtsp gates a debounced server sync (~650 ms).

LiveView state:
- Layout mode → AsyncStorage key @argus/liveview_layout_mode_v1, default "grid".
- hiddenFeedIds → in-memory only (UI filter, not persisted).
- Manage Feeds drag-to-reorder is visual hint only; reorder is not yet persisted.

Chat:
chat-context backed by /chat REST endpoints; thread list + messages drive ChatInbox / ChatThread. Permissions enforced server-side; guests can only chat with operators / admins, never other guests. New messages bump unread counters per role.

AI assistant:
ArgusAIScreen calls ai-service which proxies to the backend ai-chat endpoint. The runtime contract is in ai-chat.runtime.prompt.txt: response MUST be a single JSON object with type ∈ {text, status, mission_plan}, plus message and data. mission_plan ids must be from available_missions; status keys from drone_state.

Voice chat:
voice-chat-context + voice-chat-history persist a short voice session log; uses NSMicrophoneUsageDescription + NSSpeechRecognitionUsageDescription on iOS.

Location:
lib/location.ts (LocationPermissionStatus = granted | denied | undetermined | blocked). Used by MapScreen. iOS Info.plist NSLocationWhenInUseUsageDescription is filled. Android manifest has ACCESS_FINE_LOCATION + ACCESS_COARSE_LOCATION.
